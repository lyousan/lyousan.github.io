移动端自动化 Appium & Java

发表于2021-06-20|更新于2021-06-20
|字数总计:1.1k|阅读时长:3分钟|阅读量:

介绍

Appium 是一个开源工具，用于自动化 iOS 手机、 Android 手机和 Windows 桌面平台上的原生、移动 Web 和混合应用。通俗的讲，它可以理解为移动端平台上的selenium。只不过selenium主要针对于Web领域的测试，而Appium主要针对于移动端的测试，同样支持多种编程语言，同样都是以模拟用户的操作作为核心逻辑。

环境搭建

  • Java环境

    作为Java程序员，关于jdk的安装与环境配置这里就不具体演示了，建议安装jdk1.8的版本，相对更加稳定。

  • Android SDK

    这里演示的平台是Android，所以我们还需要Android SDK

    官网下载地址：https://developer.android.com/studio

    国内下载地址：AndroidDevTools - Android开发工具 Android SDK下载 Android Studio下载 Gradle下载 SDK Tools下载

    然后选择对应的版本下载，windows下载.exe这个版本即可

    安装后还需要配置一下环境变量

    ANDROID_HOME=安装路径

    在Path中添加**%ANDROID_HOME%\tools和%ANDROID_HOME%\platform-tools**

    打开安装后的Android SDK Manager，然后根据需要下载相应的工具，默认会勾选上一些安卓镜像，这些镜像文件会比较大，如果我们只是做测试的话可以不用下载。

    下载好之后，可以使用Win + R键，然后输入cmd 打开命令行控制台，输入adb version检验Android SDK环境是否正常。

  • Appium服务端

    Appium是基于服务端与客户端的架构，也因此可以支持各种计算机语言，入门建议使用Appium Desktop，界面化的操作会更加容易上手，并且不用单独安装node.js的环境，后期感兴趣的话可以再尝试使用Appium Server，以命令行的形式操作。

    官网地址：Appium: Mobile App Automation Made Awesome.

    github地址：GitHub - appium/appium-desktop: Appium Server and Inspector in Desktop GUIs for Mac, Windows, and Linux

    百度网盘：[提取码：autt]](https://pan.baidu.com/share/init?surl=xY8WKyEFQWllaWJNS2O_HQ)

  • 真机连接/模拟器

    这里以华为手机为例

    1. 进入设置

    2. 关于手机

    3. 一直点击版本号，直到出现 已处于开发者模式 的提示为止

    4. 进入开发人员选项，开启USB调试

    5. 使用USB连接手机到电脑，如果出现连接提示，选择以文件传输模式，并同意调试

    6. 在cmd控制台使用adb devices查看手机的udid(之后在通过appium连接手机时需要用到这个)

Hello Appium

  • 连接上真机后，打开appium，直接启动即可

    可以设置成中文，便于操作

    点击右上角的搜索按钮，进入审查模式，创建一个与手机的会话即可获取手机app中的界面元素

    关于Capabilities参数的配置：预期功能 - Appium

    第一次连接会在手机上自动安装3个软件

总结

appium是在selenium的基础上拓展了移动端上的特有实现，所以appium的使用上跟selenium上很像，不过在环境搭建上相对会比较繁琐。

同样的，在使用appium时，代码中也要时刻注意当前界面的上下文，虽然移动端的app并没有像浏览器中显式的展现出不同的窗口和标签页，但是仍然存在页面的切换，这个时候需要重新创建一个AndroidDriver才能获取到新界面中的元素。

此外appium有时还可能会出现服务端socket被挂起的异常，可以将自动安装的Appium Settings卸载掉，重新运行appium。
