总览

对于一名JAVA程序员来说，在日常的开发中往往都离不开对集合的使用，集合能够使我们在操作数据的时候更加便捷，提升开发效率。同时，集合也是面试中老生常谈的话题，几乎每个面试官都会问到集合相关的知识点。大部分的时间里我们往往都是API调用工程师，直接使用集合提供的方法去操作数据，很少会去想它底层是如何实现的，本文将详细分析我们常用的集合底层是如何实现的，以及各个集合的优缺点和使用场景。

什么是集合

集合类是Java数据结构的实现。Java的集合类是java.util包中的重要内容，它允许以各种方式将元素分组，并定义了各种使这些元素更容易操作的方法。Java集合类是Java将一些基本的和使用频率极高的基础类进行封装和增强后再以一个类的形式提供。集合类是可以往里面保存多个对象的类，存放的是对象，不同的集合类有不同的功能和特点，适合不同的场合，用以解决一些实际问题。

简单的来说，集合就是用来存放数据的，从这点上讲，这与数组的作用没有什么两样，其本质都是用于在程序中存储数据的。

为什么要使用集合？

上文中也提到从使用的目的来讲，集合与数组无异，都是为了存储数据，那为什么还要使用集合呢。

首先我们可以先来看看数组都那有些特点：

特点 影响
数组的长度不可变 数组长度设置小了不够用，大了又浪费空间
数组的内存空间是连续的 对于内存空间的要求高，必须是连续的空间
数组中只能存放同一种类型 对数据要求高，使用上不够灵活
支持随机访问 使用下标访问速度快，时间复杂度为O(1)

可以看出，数组的优点就是速度快，这里的快暂时指的是广义上的快，不具体指查询、添加、删除等操作，而缺点呢也很明显，那就是不够灵活，而集合的出现就弥补数组的缺陷。

但是这里要注意的是，数组可以存放基本数据类型和引用数据类型，但是集合只能存放引用数据类型，虽然我们平时在开发中也能直接向集合中添加基本数据类型，那是因为JAVA的自动装箱机制，帮我们把基本数据类型包装成了引用类型。

集合的实现

本章节默认读者对数组和链表这两种数据结构有一定的了解

我们先由上自下地介绍一下集合类的几个主要的接口，方便大家对整个集合体系有一个大致的理解

Iterable接口

Implementing this interface allows an object to be the target of * the “for-each loop” statement.

这是官方对于Iterable接口上的注释，翻译过来就是 实现此接口，允许对象成为” for-each”循环的目标。也就是说，实现Iterable接口后，就具备了迭代器遍历元素的能力，从编码角度讲，实现这个接口的类就能使用增强for循环遍历元素。

Collection接口

The root interface in the collection hierarchy. A collection represents a group of objects, known as its elements. Some collections allow duplicate elements and others do not. Some are ordered and others unordered. The JDK does not provide any direct implementations of this interface: it provides implementations of more specific subinterfaces like Set and List. This interface is typically used to pass collections around and manipulate them where maximum generality is desired.

同样在官方的注释中我们也可以了解到，Collection接口是作为整个集合框架体系中的根接口，不同的集合有着不同的特性，有的集合允许重复的元素，而有的不允许，有的集合是有序的，而有的是无序的，因此JDK并没有提供Collection接口的直接实现，而是根据不同集合的特性提供了更细粒度的子接口的实现，如Set，List等子接口。

Collection接口既然作为根接口，那么相应的一定定义了很多对于数据的通用操作：如添加，删除，修改，遍历等方法，不同的实现类会有各自不同的实现。不过这个根接口一般我们不会直接使用，只有在一些需要最大通用性的场景下才直接使用Collection接口。

List接口

有序集合（也称为列表 ），用户可以精确控制列表中每个元素的插入位置，可以通过整数索引（列表中的位置）访问元素，并搜索列表中的元素。

List接口的特点就是：有序且允许元素重复，可以根据索引（下标）来操作元素

ArrayList

前置的一些了解终于结束了，接下来就开始对集合框架中的实现类具体讲解。

首先是ArrayList，从类名上不难发现，这是个由数组实现的List集合，那当然也继承了数组的优点——支持下标的随机访问，同时还解决了数组的不灵活性，本小章主要探讨的就是关于ArrayList如何解决数组的不灵活性。

ArrayList如何解决数组的不灵活性

小结

ArrayList的特点：

  • 底层的数据结构是数组，查询和修改的效率高，增加和删除的效率相对较低（需要根据具体情况）

  • 能够存储null值

  • 元素是可以重复的

  • 线程不安全，但效率高

  • 有索引，能够方便的检索元素

对于ArrayList如何解决数组的不灵活性我们已经分析完了，从中涉及到源码部分的核心在添加上，因为扩容只在添加时出现，其余的操作都比较简单。

ArrayList可以存放任何类型的元素，包括null，使用中我们需要注意的是对于ArrayList来讲主要的性能消耗在于拷贝扩容上，所以使用时尽量预估好元素的个数，在初始化的时候给定构造器一个初始容量，以降低扩容的次数，从而提升性能。

同时ArrayList并不是线程安全的，虽然JDK也提供了线程安全的List，如Vector，但Vector是通过直接在方法上加synchronized关键字实现的，锁的粒度太大，性能消耗较大，并不建议使用Vector。所以在并发情况下，需要程序员自己去处理线程安全的问题。

ArrayList适合用于需要通过下标随机访问的场景，更适合用于做查询，不适合用于需要大量增删，特别是对第一个元素的增删的场景，不过在你不知道具体该使用什么样的List集合时，80%的情况下盲选ArrayList都不会有大问题。

LinkedList

同理，从类的命名上可以看出LinkedList是链表实现的List，采用链表的结构实现，天然地解决了数组中对于内存空间连续的要求，也具备了链表的优缺点。

优点 缺点
各个元素之间以保存前后节点的引用来作为连接，不要求内存空间的连续 每个节点都额外保存前后节点的引用，会占用一部分的内存空间
增删元素很便捷，只需要修改前后节点的引用即可，对于头尾元素的操作时间复杂度为O(1) 检索元素慢，需要从头节点开始遍历，对与指定位置的操作会有影响

因为LinkedList是从数据结构层面去解决的数组的缺点，在源码的实现上就不再过多的分析，这里主要再简述一下LinkedList的特点：

  • 底层的数据结构是一个双向链表（非循环链表），可存放任意类型的元素，包括null，且元素有序可重复

  • 在进行数据的添加和删除操作时效率高，查询和修改效率低，顺序访问会非常高效，而随机访问效率会比较低

  • 实现了List接口，支持使用索引访问元素，但由于链表的结构，数据量较大时检索速度会变慢

  • 实现Deque接口，所以LinkedList也可以当做双端队列使用

  • 是线程不安全的，但效率高

对于LinkedList不要只知道add()方法可以添加元素，还有一些对应双向链表特性的API，如push(),pop(),peek(),poll()等等，同时LinkedList更适合用于一些只针对头尾节点操作的场景，如队列，栈等，不适合在需要频繁检索和指定位置操作的场景使用。
